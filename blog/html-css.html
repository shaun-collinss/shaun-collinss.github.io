The Question of classes vs Id's

I looked over this question as I think its a very important one to be able to understand. When I first started doing some HTML I didnt quite understand what the difference was. This can become a huge problemn if think about it for to long. Especially when starting out. The best way to describe them would be thinking about your intent and what you want to accomplish. Before adding either a class or an ID selector you should ask yourself whether this item will be unique or if it will be similar to other items. This is a good question to ask when deciding which one to use as it can save you a lot of hassle in the long run. A great example for a class would be something that would come up numerous times throughout production. A great real world example would be a cheesecake shop, because who doesnt really like cheesecakes. A class would be similar to a type of cake available in the shop. You can have a cheesecake or a torte cake. Each of them would be considered different classes of cakes. When you buy a cheesecake you know it will have cream cheese in it and a crust, toppings could be different depending on the type of cake. The use of ID's on the other hand would be more similar to specialty items. Items that don't quite fit a class and are more likely to only be one of them or very few. In regards to this the cheesecake shop analogy this would be like having a keto friendly cheesecake. A cheesecake that would be unique because of the way it is made and the ingredients that would be used. It's unique from the standpoint of it being diet specific and for the most part is made differently
