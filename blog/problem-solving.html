Why is Code readability is important?

The purpose of making your code readable is so that others that come along are able to read your code more easily. Thus do not have to spend long periods trying to work out what each portion of your code does. This also makes it easier for code to be edited when needed. Identation is a nice and easy way to make your code readable outside of adding comments.

What is DRY?

DRY - Don't repeat yourself

What this means is that where possible you should refractor your code into a function or method to be recalled when needed rather than writing out new code each time. The greatest reason why you would do this is that if you need to change something instead of going and changing each individually which would consume a lot of time you could just change a single method which would change it throughout the rest of your code.
